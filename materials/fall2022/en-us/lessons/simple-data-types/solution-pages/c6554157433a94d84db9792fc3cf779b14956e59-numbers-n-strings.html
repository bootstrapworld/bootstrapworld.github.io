Numbers and Strings

Make sure you’ve loaded the code.pyret.org (CPO), clicked “Run”, and are working in the Interactions Area.

Numbers

1 Try typing 42 into the Interactions Area and hitting “Enter”. What is the largest number the editor can handle?

I didn’t find a number that was too big!

2 Try typing 0.5. Then try typing .5. Then try clicking on the answer. Experiment with other decimals. Explain what you understand about how decimals work in this programming language.

In pyret, decimals need to be typed with a zero in front of the decimal point.

3 What happens if you try a fraction like 1/3? Pyret will return the decimal equivalent. It even does repeating decimals!

4 Try writing negative integers, fractions and decimals. What do you learn?

Pyret can handle negatives.

Strings

String values are always in quotes.

5 Is 42 the same as "42"? Why or why not? Write your answer below:

No. 42 is a number and "42" is a string.

  • Try typing your name (in quotes!).

  • Try typing a sentence like "I’m excited to learn to code!" (in quotes!).

  • Try typing your name with the opening quote, but without the closing quote. Read the error message!

  • Now try typing your name without any quotes. Read the error message!

6 Explain what you understand about how strings work in this programming language. Strings require quotes.

Operators

7 Just like math, Pyret has operators like +, -, * and /. Try typing in 4 + 2, and then 4+2 (without the spaces). What can you conclude from this?

Operators require spaces.

8 Type in the following expressions, one at a time: 4 + 2 * 6, (4 + 2) * 6, 4 + (2 * 6). What do you notice? If you use more than one operator you have to insert parentheses to tell the computer which operation to apply first.

If you use more than one operator you have to insert parentheses to tell the computer which operation to apply first.

9 Try typing in 4 + "cat", and then "dog" + "cat". What can you conclude from this?

You can combine two strings, but you can’t combine a number and a string.
